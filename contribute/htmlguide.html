Element Description Attributes Children
d-box Text box
  • theme: Changes the border. See below for names/descriptions of available themes.
  • If small dialogue, then optionally place; may have one or more of the following values, separated by spaces
    • right (default), left, or center-x
    • bottom (default), top, or center-y
    • center (shorthand for center-x center-y)
  • If dialogue: either d-sprite or d-speaker
  • At least 1 d-text
  • If including small dialogue: d-box
d-speaker Speaker label None Plaintext label (e.g. Susie)
d-sprite Speaker label with sprite
  • speaker: Plaintext label
  • variant: If speaker has multiple portrait sets, path to specific portrait set
  • emotion: The code for this specific portrait
 None
d-text A line of text
  • asterisk: To include a * before the text. Can be set to one of the following to change the asterisk to a group speaker's symbol (case-insensitive):
    • Sweet
    • Cap'n or Capn
    • K_K or KK
    • Happy or Nice
    • Mad or Mean
    • Sad or Shy
  • font = "tiny" | "sans" | "8bit": Changes the font used to Noelle's shy whispering, Comic Sans, or 8-bit, respectively.
Plaintext
d-choices Choice menu
  • theme: Changes the border. See below for names/descriptions of available themes.
  • offset: For offsetting the heart when the L/R options are too long. Positive % values to shift it right (e.g. offset="10%"), negative to shift left (e.g. offset="-10%")
  • noninteractive: If options change nothing, use this attribute and don't include d-option-panels afterwards
 d-option
d-option Option for choice menu
  • index = 0 | 1 | 2 | 3: Placement of option (corresponds to left/right/top/bottom)
  • text: Option text. Replace instances of # with the following:
    • \s for a linebreak between words
    • \b for a linebreak within a word
    • \h to hide the next character in compact mode
    Example: When talking to Onion, the options for your own name are "Kris" and "Hippo-#potamus". The latter would be formatted as <d-option index="1" text="Hippo\h-\bpotamus">. The \b means it'll be split with a hyphen in full-size mode, and the \h means it'll display as Hippopotamus without the hyphen in compact mode's single-line format.
  • conditional: For when an option only appears under specific conditions
 None
d-option-panel Output of a certain choice
  • index = 0 | 1 | 2 | 3: Corresponding option's index
  • theme: Changes the border. See below for names/descriptions of available themes.
 Pretty much anything. d-box, d-choices, d-option-panel, p.note...
<p class="note"> (a.k.a. p.note) Annotation None Text. Used for clarifying conditions or actions that happen between dialogue.

Themes

Borders for dialogue boxes/choice menus etc. Use in the format theme="light", theme="dark" etc.

light

Default value; if no theme is explicitly set (i.e. you leave out the theme="..." attribute), elements will generally use this.

Light World border, a plain white box.

dark

Dark World border, a ridged box with elaborate corners.

fade

Used for special text which doesn't have a border in-game, such as the Game Over text or the Vessel creation intro.

No border; the edges simply fade into transparency.

Battle themes: battlemenu, battletalk, battledesc

For use in encounters.

battlemenu: No border with solid edges. Any text that displays below the party members' names/HP/action menus. May have face sprites.

The "battlemenu" box. Wider than standard dialogue boxes.

battledesc: Narrower cousin of battlemenu, this is for ACT/item descriptions that appear on the bottom right side of the screen.

Battle desc box 16% TP

battletalk: White text bubble with the Dotum font. For dialogue that appears next to characters' sprites on the top half of the screen.

Tutorial The "battletalk" box.

Object themes: objmenu, objreact

For use in overworld narration/dialogue related to items.

objmenu: No border with solid edges. Object descriptions. Same width/border as battlemenu, but uses a condensed font variant.

An object's description.

objreact: No border with solid edges, but not as tall and using simple party icons instead of face sprites. For party members' responses to using items or equipping weapons/armor.

Susie Character's reaction goes here.

Shop themes: shopleft, shopright, shopdesc

Dark World border with special sizing.

shopleft: for the initial dialogue that appears when you first enter a shop. The wider variant.

Welcome to shop.

shopright: for dialogue that appears on the right side, once you start clicking into menus. The narrower variant.

Buying something?

shopdesc: for descriptions of items on sale. Same width as shopright, but uses the condensed font and has slightly different spacing.
