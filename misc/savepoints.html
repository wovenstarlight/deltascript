SAVE menu layout

Upon interacting with a SAVE point, preceding narration (if any) plays; see the below sections for this.

After any narration has played, the initial SAVE menu opens, from which you can choose to SAVE, view additional menus (Chapter 2 onwards only), or close the menu. This initial menu lists the following:

  • Name: The owner of the existing SAVE file in the current slot (if any), represented in all examples below as "<NAME>".
  • The current LV, represented below as "<LV #>" where # is typically the current Chapter number (with an exception in Chapter 1).
  • Time: the duration for which you've been playing on the current file, with the seconds updating in real time. (It is not the time at which the last SAVE in the current slot was made!) Typically displayed in hours, minutes, and seconds, starting at "0:00:00", and represented below as "H:MM:SS".
    • Prior to Chapter 1&2 Demo v1.16, in Chapters 1 & 2, the timer displayed only minutes and seconds, starting at "00:00", with minutes capable of going into 3+ digits.
  • Location: represented below as "<Location>".

Chapter 1

Initial

<NAME> LV # H:MM:SS <Location> Save Return

In the above menu:

  • "<NAME>" depends on the last existing SAVE file in the current slot:
    • "Kris" if no SAVE has been created/erased in this slot before
    • "[EMPTY]" if a SAVE has been erased in this slot before
    • the player's true name (chosen during the vessel creation sequence) if a SAVE file currently exists in this slot i.e. if you've SAVEd at least once
  • LV depends on the last existing SAVE file in the current slot:
    • "LV 1" on a SAVE belonging to Kris or the player
    • "LV 0" if a SAVE has been erased in this slot before
  • "<Location>" is the location where the last existing SAVE file in the current slot was created. If no SAVE file exists for this slot yet, this instead displays as "---".

Upon selecting "Save"

The SAVE chime plays.

<NAME> LV 1 H:MM:SS <Location> File saved.

The name, level, and location are all updated; the player's true name and the current SAVE point's location are used. The timer continues ticking. Pressing [CANCEL] at this point closes the menu, returning you to the overworld.

Chapter 2 onwards

Initial

<NAME> LV # H:MM:SS <Location> Save Return Storage Recruits

In the above menu:

  • "<NAME>" is the player's true name, either chosen during Chapter 1's vessel creation sequence (if a SAVE has been continued from Chapter 1) or chosen immediately before gameplay begins (if starting a fresh playthrough from a subsequent chapter).
  • LV is the current Chapter's number.
  • "<Location>" is the current SAVE point's location.
  • Selecting "Storage" opens the storage menu.
  • Selecting "Recruits" opens the recruit menu.

Upon selecting "Save"

A menu opens, showing live data for the current playthrough at the top, below which is data for the three SAVE slots which can be navigated through. By default, the cursor starts on the SAVE slot currently being played in.

Below is a sample menu showing one existing SAVE file and two empty SAVE slots, where the player is playing in SAVE slot 2:

LV # <NAME> H:MM:SS <Location> LV # <OTHER NAME> H:MM:SS <Other Location>
New File
New File
Return

Choosing "Return" at this point returns you to the initial menu.

Upon choosing a slot to SAVE in

The SAVE chime plays. The top section turns yellow and its timer freezes at the moment [CONFIRM] was pressed, so that the top section shows precisely the values saved to file. The chosen slot has all text replaced by a label reading "File Saved". The "Return" option at the bottom disappears.

Below is the sample menu, now with the player having chosen to SAVE in slot 2:

LV # <NAME> H:MM:SS <Location> LV # <OTHER NAME> H:MM:SS <Other Location>
File Saved
New File

Pressing [CANCEL] at this point closes the whole menu, returning you to the overworld.
